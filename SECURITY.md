# Security Policy

## Supported Versions
The following versions of ActiveMQ Classic 6 receive security updates:
- Latest stable release
- `main` branch

Older versions do not receive security patches.

## Reporting a Vulnerability
To report a security issue:
1. Do not open a public issue.
2. Contact the maintainers privately via email or a designated secure channel.
3. Include:
   - Detailed description
   - Steps to reproduce
   - Potential impact
   - Any proposed fixes or patches

## Vulnerability Handling Process
Maintainers will:
- Acknowledge the report within 72 hours
- Assess severity and impact
- Prepare a fix or mitigation
- Coordinate responsible disclosure

## Disclosure
Security issues are disclosed publicly only after:
- A fix is available
- Major users have been notified
